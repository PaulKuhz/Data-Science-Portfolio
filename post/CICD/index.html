DevOps: Automating CI/CD for data engineering workflows using Azure, Terraform, and DataFactory

September 15, 2024

In this project, I develop a full CI/CD pipeline tailored for data engineering using Azure and Terraform for infrastructure-as-code in PyCharm. It covers establishing an Azure resource group, setting up blob storage with source and destination containers, and constructing an Azure Data Factory pipeline for data transfer. I manage infrastructure setup through Terraform and Azure CLI, while automated testing validates data migration using Azure Data Factory.

System Architecture

Starting from the root entry point, I created a resource group and deployed two resources: a storage account and a data factory. The first module includes a storage account with blob storage and a storage container, alongside both a source container and a destination container.

For the data pipeline, I created a Copy Data activity that includes a linked service, a data factory binary service, and the data factory engine itself. Additionally, a 'data' folder is created to temporarily store files. An overview of the system architecture is provided below:

Creating the Azure Resource Group and Storage Account

At first I create an Azure resource group and the storage account using Terraform. This included defining the variables for the resource group name, location, and tags, as well as the storage account name, replication type, and environment tags.

The Storage Container Resource utilizes a for_each loop to generate multiple containers based on var.source_folder_name and var.destination_folder_name. Each container's name is determined by the for_each key and is associated with the storage account, while its access level is defined by a variable. The Storage Blob Resource creates a blob named test.txt within the source container, specifying its type as Block. Finally, the Output section provides the primary access key for the storage account, allowing for programmatic access.

Deploying the Azure Data Factory

I then developed a Terraform script to set up Azure Data Factory (ADF) for transferring binary data from a source storage blob to a destination storage blob. I created an ADF resource with a system-assigned managed identity and fetched details for both the source and destination storage accounts based on the provided names and resource groups. I established two linked services connecting the ADF to each Azure Blob Storage account.

I defined datasets for specific folders/files in the source, including a file named test.txt, and in the destination storage blobs, where the destination used a dynamically generated filename featuring a timestamp. I outlined a data copy activity in the pipeline, which included a copy activity for data transfer, BinarySource and BinarySink types for handling binary data, a daily scheduler, and links to the source and destination datasets for data ingestion and storage.

I then focused on deploying the ADF by setting up the data factory, configuring linked services for both storage accounts, defining datasets, and creating the copy data pipeline with policy and scheduler settings to automate data transfer.

Testing & Validation

After testing and validating the deployed infrastructure, which included checking the storage account containers, testing the linked service connections, and running the copy data pipeline, I successfully demonstrated the data migration from the source to the destination container.

Key steps of this system architecture design, were Azure storage setup, deployment automation, and continuous testing integration, supporting a streamlined, modular, cloud-based data engineering workflow.
